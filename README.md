## Разработка грамматики

Грамматика - G = {V_t, V_n, P, Z}
### ```V_t = { for, in, range, print, id, num, :, ;, (, )}```
### ```V_n = { <program>, <for>, <range>, <print>, <expr>, <block>} ```
### P:
```
1) <program> -> <for>
2) <for> -> 'for' 'id' 'in' <range> ':' <block>
3) <range> -> '(' <expr> ')'
4) <block> -> <print> | <for> |
5) <print> -> 'print' '(' <expr> ')' ';'
6) <expr> ->  num | id
```
### ```Z = {<print>}```

## Классификация грамматики (по Хомскому)

Моя грамматика является контекстно-свободной
  Обоснование:
- Левая часть каждого правила — один нетерминал 
- Правая часть может содержать любую последовательность терминалов и нетерминалов, без ограничений на контекст.
- Грамматика не является регулярной, так как в правых частях встречается более одного нетерминала и терминалы не только в начале/конце.
- Отсутствуют правила с контекстной зависимостью (левая часть длиннее одного символа).

## Метод анализа (алгоритм синтаксического анализа - граф автоматной грамматики или рекурсивный спуск).

В данной работе реализован синтаксический анализатор (парсер), построенный на основе метода рекурсивного спуска. Функции вызывают друг друга рекурсивно в соответствии с правилами грамматики, имитируя левый вывод.

### Особенности реализации:

Грамматика приведена к LL(1)-виду, что позволяет вести детерминированный разбор без возвратов.

Функция Match сравнивает текущий токен с ожидаемым; при несовпадении фиксирует ошибку.

Для нейтрализации синтаксических ошибок используется метод Айронса (паническое восстановление): при ошибке парсер пропускает токены до ближайшего «синхронизирующего» ключевого слова (for или print), после чего продолжает анализ. Это позволяет обнаружить все ошибки за один проход.
